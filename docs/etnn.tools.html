calc_and_collect(config_id: str, epoch: int, train_y_true: Tensor, train_y_pred: Tensor, train_loss: Tensor | None = None, test_y_true: Tensor | None = None, test_y_pred: Tensor | None = None, test_loss: Tensor | None = None)[source]

This function calculates and collects various metrics for a given configuration and epoch. It writes the results to a CSV file specified by the attribute storage_total_path.

The function uses the attributes regression_metrics, classification_metrics, and mode_order to determine which metrics to calculate and in which order. It also uses the attribute is_classification to check if the problem is a classification or a regression problem.

The function appends a new row to the CSV file with the following format: config_id, epoch, train_loss, test_loss, train_regression_scores, test_regression_scores, train_classification_scores, test_classification_scores.

If any of the values are missing or None, they will be written as 0. If any of the modes are not provided, they will be skipped and filled with 0s.

Parameters:

  • config_id (str) – The identifier of the configuration.

  • epoch (int) – The number of the epoch.

  • train_y_true (torch.Tensor) – The true labels for the training set.

  • train_y_pred (torch.Tensor) – The predicted labels for the training set.

  • train_loss (torch.Tensor) – The loss value for the training set. If None, it will be written as 0.

  • test_y_true (torch.Tensor) – The true labels for the test set.

  • test_y_pred (torch.Tensor) – The predicted labels for the test set.

  • test_loss (torch.Tensor) – The loss value for the test set. If None, it will be written as 0.

Returns:

Nothing currently

Return type:

None
