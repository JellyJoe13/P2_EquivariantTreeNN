Function that allows for the generation of a very simple dataset that is more symbolic than actually usable. Uses simple integers ranging from 0 to #elements spanned by tree and adds labels to them according to their permutation group. Assigns one permutation tree per label.

Parameters:

  • permutation_tree (TreeNode) – Tree containing the struction of a permutation tree which in itself determines which elements to consider equal in a set input setting.

  • integer_generation (bool) – Determines if integer are to be generated by the function. Currently only functionality for the case True is implemented. Default: True.

  • num_classes (int) – Defines the number of classes (and hence also the number of permutation trees) to be contained in the returned data. If not enough permutation trees are available it may happen that classes are empty. Default: 2.

Returns:

Tuple of data and corresponding classes

class etnn.data.ferris_wheel.FerrisWheelDataset(df_health, df_index, num_gondolas)[source]

Bases: Dataset

Pytorch dataset of the ferris wheel dataset concept. Each element in the dataset consists of data x and a label y.

y in this case is the happyness score calculated by the function build_wheel_happyness(...).

x consists of gondolas which contains a certain number of people, which is specified by the input dataset. The logic how many gondolas and persons per gondolas are is not explicitly contained in this object but contained in the element df_index through column names. The tensor provided by this function however will not be in any specific order or structure that indicates which data is assigned to which gondola.

etnn.data.ferris_wheel.load_modified_ferris_wheel_dataset(num_gondolas: int = 10, num_part_pg: int = 5, num_to_generate: int = 1000, num_valid_to_add: int = 1000, num_invalid_to_add: int = 1000, df_name_input: str = 'Sleep_health_and_lifestyle_dataset.csv', dataset_path: str = 'datasets', df_intermediate_output_name: str = 'health_dataset_preprocessed-1.csv', try_pregen: bool = True, seed: int = 4651431, label_type: str = 'default', final_label_factor: float = 0.001) → Tuple[FerrisWheelDataset, DataFrame][source]

Function that loads dataset from pre-generated csv files or generates data(and dataset) (and saves these intermediate csv files). Loads a dataset that not exactly follows the allowed permutations combined with the exact calculated happyness scores. It contains these items in the beginning and then adds permutated pure items to increase the potential benefit of equivariant nn’s but afterwards adds label perturbed data.

Parameters:

  • num_gondolas (int) – Number of gondolas the ferris wheel should have, default: 15

  • num_part_pg (int) – Number of persons in each gondola, default: 5. (by default no empty passengers/seats will be generated)

  • num_to_generate (int) – Number of dataset entries to generate, default: 1000

  • num_valid_to_add (int) – Number of pure items to add which (permuted) already exist within the original dataset, default: 1000

  • num_invalid_to_add (int) – Number of perturbed items to add to the previously pure data to challenge the neural networks capability to react to falsified data, default: 1000

  • df_name_input (str) – Name of the dataset to use as the input for the persons health data, default: Sleep_health_and_lifestyle_dataset.csv

  • dataset_path (str) – Specifies the path to the dataset folder, default DEFAULT_DATA_PATH

  • df_intermediate_output_name (str) – Name of the intermediate datasets output, default: 'health_dataset_preprocessed-1.csv'. Intermediate dataset denotes the preprocessed health dataset.

  • try_pregen (bool) – Controls whether pre-generated shall be loaded or new data should be generated hence overwriting and updating previously generated data with the same parameters, default: True

  • seed (int) – Seed to use for random generation

  • label_type (str) – label to be generated. Viable options: 'default' producing the most complex label with logic inspired by a real ferris wheel, 'tree' and 'tree_advanced' for a tree based generation of label with minimal logic behind this label. Difference in first and second tree option is that the first option is a downgraded version only using S and P type labels and not C type labels instead of C as it would have been intended for a ferris wheel dataset, default: 'default'.

  • final_label_factor (float) – Factor by which the label is scaled. Does not apply for 'default' option in label_type parameter. Default: 1/1000

Returns:

generated dataset and index dataframe for data loading purposes

Return type:
